# CLAUDE.md

## Project
Python calculator API — a simple but production-ready project used to demonstrate the PETER framework and AI Developer Workflows (ADWs).

## Architecture
- `src/calculator.py` — Core calculator module (pure functions)
- `src/api.py` — FastAPI REST API wrapping calculator functions
- `tests/test_calculator.py` — Unit tests for calculator module
- `tests/test_api.py` — API integration tests
- `adws/` — AI Developer Workflow scripts (autonomous SDLC pipeline)
- `specs/` — Implementation plans generated by planning agents
- `.claude/commands/` — Slash command templates for specialized agents

## Commands
- Run unit tests: `python3 -m pytest tests/test_calculator.py -v`
- Run API tests: `python3 -m pytest tests/test_api.py -v`
- Run all tests: `python3 -m pytest tests/ -v`
- Start API server: `python3 -m uvicorn src.api:app --reload --port 8000`

## Conventions
- All functions require type hints, docstrings, and tests
- Raise `ValueError` for invalid inputs (not silent failures)
- Follow existing patterns in calculator.py for new math functions
- API endpoints follow REST conventions: POST for operations
- Specs go in `specs/` with format: `issue-{number}-{description}.md`
- One function per PR, keep changes focused

## ADW System
The `adws/` directory contains the autonomous workflow pipeline:
- `adw_plan.py` — Generate implementation plan from GitHub issue
- `adw_build.py` — Implement the plan
- `adw_test.py` — Run tests and fix failures
- `adw_review.py` — Code review
- `adw_sdlc.py` — Full SDLC pipeline (plan → build → test → review)

## Quality Gates
- All tests must pass before PR merge
- Every new function needs ≥3 test cases
- Error handling required for edge cases (division by zero, negative sqrt, etc.)
